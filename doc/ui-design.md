# Soloist Cabin Pro — UI 设计文档

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **像素风 UI** | 全局采用像素艺术风格：阶梯边框、像素字体、色块按钮、16px 网格对齐 |
| **多巴胺配色** | 明黄主色，高饱和高对比辅色，适合像素风的大色块表现 |
| **像素角色陪伴** | 5 只像素小动物（16×16 网格），在不同场景出现 |
| **信息渐进披露** | 打卡页只展示当前时段，历史页默认周视图 |

---

## 2. 配色方案

像素风适合高饱和、大色块、强对比。所有颜色均从 8-bit/16-bit 调色板精选，确保在像素渲染下辨识度高。

### 2.1 主色板

```
主色      明黄    #FFE030  ── 导航栏、主按钮、强调元素
主色暗    琥珀    #E0A800  ── 按钮阴影/暗面 (像素 3D 效果)
背景      奶油    #FFF8E8  ── 页面底色
卡片      纯白    #FFFFFF  ── 卡片/弹层背景
卡片暗面  浅米    #F0E8D0  ── 像素边框暗面
文字      深棕    #3A3028  ── 正文 (暖黑代替纯黑)
文字浅    灰褐    #8A8078  ── 辅助文字
阴影      纯黑    #000000  ── 像素阴影 (1-2px 偏移)
```

### 2.2 多巴胺辅色 (像素调色板)

所有辅色均配亮面/暗面两个色阶，用于像素按钮的 3D 凸起效果：

| 色系 | 亮面 | 暗面 | 用途 |
|------|------|------|------|
| 珊瑚粉 | `#FF6B8A` | `#D94A6A` | 迟到/旷工/惩罚 |
| 薄荷绿 | `#50E8B0` | `#30C090` | 全勤/奖励/完成 |
| 薰衣草 | `#B090F0` | `#9070D0` | 请假/拍摄日 |
| 天空蓝 | `#60C8FF` | `#40A0E0` | 正常出勤/准时 |
| 暖橙 | `#FF9040` | `#E07020` | 加班/超额 |
| 西瓜红 | `#FF5070` | `#E03050` | 严重警告/旷工 |

### 2.3 功能语义色

像素风不使用淡色半透明背景，改用**纯色小块 + 粗边框**：

| 场景 | 色块色 | 边框色 | 图标色 |
|------|--------|--------|--------|
| 正常出勤 | `#E0F4FF` | `#60C8FF` | `#60C8FF` (蓝) |
| 迟到 | `#FFE8EC` | `#FF6B8A` | `#FF6B8A` (粉) |
| 早退 | `#FFF0E8` | `#FF9040` | `#FF9040` (橙) |
| 旷工 | `#FFE0E8` | `#FF5070` | `#FF5070` (西瓜红) |
| 请假 | `#F0ECFF` | `#B090F0` | `#B090F0` (紫) |
| 拍摄日 | `#FFF8E0` | `#FF9040` | `#FF9040` (橙) |
| 已完成 | `#E0FFF0` | `#50E8B0` | `#50E8B0` (绿) |

### 2.4 像素边框与组件风格

所有 UI 组件采用**阶梯边框**代替圆角，通过 2-4px 的错位矩形实现像素凹凸感：

```
像素凸起按钮 (浅边框=亮面, 深边框=暗面):
  ┌──────────────────────┐
  │                      │  ← 亮面 2px 边框 (顶部 + 左侧)
  │    按  钮  文  字    │
  │                      │  ← 暗面 2px 边框 (底部 + 右侧)
  └──────────────────────┘

像素卡片:
  ┌──────────────────────┐
  │                      │
  │    卡片内容           │
  │                      │
  └──────────────────────┘
    └─ 底部 2px 暗色阴影 (黑色，向右下偏移 2px)

像素输入框:
  ┌──────────────────────┐
  │                      │  ← 内凹效果 (暗面在顶部+左侧)
  │    输入文字           │
  └──────────────────────┘
```

组件通用规则：
- 边框宽度统一 **2px**（粗线强化像素感）
- 无圆角，全部直角
- 所有间距对齐 **8px 网格**（按钮高 48px、卡片内边距 16px 等）
- 阴影用纯黑 `#000`，`x=2px y=2px`，不透不模糊

## 3. 导航结构

```
┌─────────────────────────────────┐
│         内容区域                 │
│         (页面切换)               │
│                                  │
│                                  │
├─────────────────────────────────┤
│  📋打卡  │ 📅历史 │ 🎯对赌 │ ⚙️设置 │  ← 底部导航栏
└─────────────────────────────────┘
```

底部导航栏 4 个 Tab：

| Tab | 图标 | 文字 | 页面 |
|-----|------|------|------|
| 1 | 像素印章 | 打卡 | 主界面（打卡 + 任务 + 战报入口） |
| 2 | 像素日历 | 历史 | 历史记录（周/月/年 Tab） |
| 3 | 像素靶心 | 对赌 | 每周对赌任务管理 |
| 4 | 像素齿轮 | 设置 | 参数设置（分组折叠） |

导航栏高度 56px，选中项明黄色高亮，图标使用像素风格 16×16 网格绘制。

#### 4.1.2 时段卡片交互

```
初始状态（上午）:
┌────────────────────────────┐
│  ☀️ 上午  09:00-12:00     │
│        [ 签 到 ]           │  ← 大按钮

```

动画：300ms `ease-in-out`，上推下收

#### 4.1.3 状态显示框

实时文案规则：

```
该时段已签到未签退: "工作中..."
该时段完成(正常): "正常签到 09:00 / 签退 12:00"
该时段有异常: 高亮异常标签
  迟到: "迟到 09:15 / 签退 12:00" (珊瑚粉色)
  早退: "正常签到 09:00 / 早退 17:30" (暖橙色)
  旷工: "旷工" (珊瑚粉色底)
  请假: "已请假" (薰衣草色)
  拍摄日: "拍摄中 📸" (暖橙色)
  忘记签退(自动): "正常签到 09:00 / 自动签退 18:00" (带灰色(自动)标记)
该时段未到: "等待签到..." (灰色)
非工作日: "休息日 🐼" (团团小图标)
```

#### 4.1.4 请假按钮状态

```
上午打卡时间之前:    [请假] 可点击 → 弹窗选 morning/afternoon/all_day
上午签退后~下午打卡前: [请假] 可点击 → 弹窗仅可选 afternoon
其他时间:            [请假] 灰色不可点
非工作日:            隐藏
```

#### 4.1.5 打卡成功动画

```
1. 按钮缩小 + 变为勾号 ✅ (150ms)
2. 兜兜(🐻)从右下角弹入 (300ms spring)
3. 兜兜比 ✌️，轻微左右摇摆 (2次)
4. 兜兜缩回右下角 (200ms ease-out)
5. 时段卡片收起，下一时段展开
总时长: ~2秒
```

#### 4.1.6 男友承诺输入

上午首次打卡成功后，兜兜跳出比 ✌️ 后 → 紧接着弹出承诺输入弹窗：

```
┌──────────────────────────┐
│   🐻 设定今日奖励         │
│                          │
│   如果今天工作满 8 小时    │  ← 门槛从设置读取
│   奖励自己：               │
│   ┌────────────────────┐ │
│   │ 一杯奶茶           │ │  ← 文字输入
│   └────────────────────┘ │
│   数量：[1]  +  -        │  ← 步进器
│                          │
│   [ 确定 ]  [ 跳过 ]     │
└──────────────────────────┘
```

确定后，承诺显示在打卡页男友承诺区，旺仔(🐶)出现在旁边。

### 5.2 角色视觉规范 (像素风)

所有角色使用 **16×16 像素网格** 绘制，规范如下：

```
16×16 像素网格 (每个格子 = 1 个色块):
┌────────────────┐
│................│  ← 顶部留 1px 透明边距
│................│
│..████████████..│  ← 身体主体
│..████████████..│
│..████░░██████..│  ← ░ = 亮面 (高光)
│..████░░██████..│  ← █ = 主色
│..████████████..│  ← ▓ = 暗面 (阴影)
│..██▓▓▓▓▓▓██..│
│..████████████..│
│................│
│................│  ← 底部留 1px 透明边距
└────────────────┘
```

**通用规范：**
- 画布：**16×16 像素** (最小角色) 或 **32×32 像素** (主角色/大尺寸)
- 角色在 UI 中的渲染尺寸：**64×64 dp**（16px 画布 ×4 整数倍放大）
- 单个角色颜色：**≤ 4 色** (主色 + 亮面 + 暗面 + 轮廓/特征色)
- 眼睛：2×2 像素黑块 (豆豆眼) 或 1×2 像素竖线 (眯眼笑)
- 腮红：2×2 像素浅粉方块，位于眼下外侧
- 格式：PNG (像素图用 SVG 反而麻烦，直接存按像素对齐的 PNG)
- 放大方式：**nearest-neighbor**（整数倍放大，保持像素锯齿感）

### 5.4 角色状态与动画

每个角色至少 2 帧：

| 状态 | 帧数 | 说明 |
|------|------|------|
| 待机 | 1 帧 | 静止姿态，出现在页面角落 |
| 动效 | 2-4 帧 | 简易逐帧动画（左右偏移 1px 即实现摇摆；Y 偏移 + 眼睛变化 = 跳跃） |

像素动画实现方式：按固定帧率切换 PNG 帧（类似 GIF sprite sheet），而非 SVG 变形。

### 5.5 角色 sprite sheet 格式

每个角色一个 PNG 文件，横向排列帧，文件名如 `dudu_16x16.png`：

```
兜兜 sprite sheet (16×16 每帧, 4 帧横向排列 = 64×16):
┌────┬────┬────┬────┐
│待机│跳1│跳2│✌️ │  ← 4 帧
└────┴────┴────┴────┘
```

APP 加载时一次性读到内存，按需切换帧。

## 6. 动画规范

像素风动画以**逐帧切换**为主，辅以简单的位移/缩放。

| 场景 | 动画方式 | 帧数/时长 | 说明 |
|------|---------|----------|------|
| 时段卡片展开/收起 | 高度变化 | 200ms | 像素阶梯式展开，每 8px 一步 |
| 打卡成功反馈 | 兜兜逐帧 + 位移 | 2000ms | 3 帧 (弹出→✌️→缩回) 每帧 500ms，Y 轴弹跳 |
| 任务滑动完成 | 划掉 + 旺仔侧入 | 400ms | X 轴滑入，配合旺仔 2 帧摇尾 |
| 全勤里程碑 | 团团逐帧 | 1500ms | 4 帧 (冒出→抱星→转1→转2)，每帧 375ms |
| 战报弹层 | Y 轴弹出 | 250ms | 从底部滑入 |
| 请假弹窗 | 逐帧弹出 | 200ms | 4 帧 (0→50%→80%→100% 缩放) |
| 分组折叠 | 高度变化 | 200ms | 阶梯式收起 |
| 页面切换 | X 轴推入 | 200ms | 左右滑动 |
| Tab 切换 | 渐隐渐显 | 150ms | 透明度交叉 |

逐帧动画帧率固定 **4 FPS**（每帧 250ms），保持 8-bit 游戏时代的手感。
