## 建议实施顺序

| 阶段 | 模块 | 依赖关系 |
|------|------|---------|
| 第 1 步 | **M9** 时间抽象层 + EventBus | 无依赖，基础设施最先搭建 |
| 第 2 步 | **M6** 设置模块 | 仅依赖 M9，其他模块依赖它 |
| 第 3 步 | **M1** 考勤模块 | 依赖 M9 + M6 |
| 第 4 步 | **M2** 奖惩模块 | 依赖 M9 + M6 + M1（读打卡数据） |
| 第 5 步 | **M8** 激励模块 | 依赖 M1（读出勤判定） |
| 第 6 步 | **M3** 战报模块 | 依赖 M1 + M2 + M4（只读消费） |
| 第 7 步 | **M4** 拍摄日模块 | 依赖 M9 + M6 + M1（打卡窗口判断） |
| 第 8 步 | **M5** 历史模块 | 依赖 M1 + M2（只读消费） |
| 第 9 步 | **M7** 同步模块 | 依赖所有模块（订阅事件 + 读数据） |

> 注：M3/M4/M5 之间无依赖，可并行；M8 依赖轻，可与 M2 并行。
