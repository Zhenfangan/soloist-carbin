# Soloist Cabin Pro — 需求文档 (Proposal)

## 1. 项目概述

**Soloist Cabin Pro** 是一款面向自媒体从业者的自律打卡 Android APP，
帮助用户记录每日上下班考勤、加班情况，根据出勤状态自动计算虚拟奖惩，
生成手账风格的每日战报，支持历史回顾和每周任务对赌。

**目标用户：** 1 人（单用户，本地为主）

---

## 2. 技术方案

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端 APP | Python + Kivy + KivyMD | 打包为 Android APK，Material Design 风格 |
| 数据本地存储 | SQLite | 手机本地数据库，离线可用 |
| 后端同步服务 | Python FastAPI | 部署于云服务器 |
| 检阅端 | 网页 + 微信/飞书机器人 | 网页查看完整记录，战报生成时通过机器人推送至男方微信/飞书 |
| 实时推送 | WebSocket | 打卡、战报生成时实时推送至后端 |

## 3. 功能模块

### 3.1 考勤打卡

每天分为三个时间段：

| 时段 | 打卡类型 | 说明 |
|------|---------|------|
| 上午 | 上班打卡 + 下班签退 | 必须，时间由用户自行设置 |
| 下午 | 上班打卡 + 下班签退 | 必须，时间由用户自行设置 |
| 晚上 | 加班签到 + 加班签退 | 可选，下午签退后弹出提示 |

### 3.2 出勤状态判定

系统根据用户设置的标准时间，自动判定每次打卡的状态：

| 状态 | 判定规则 |
|------|---------|
| **正常** | 在设定的上班时间之前打卡 |
| **迟到** | 晚于设定的上班时间打卡 |
| **早退** | 早于设定的下班时间签退 |
| **请假** | 在上下班打卡时间之前手动点击请假按钮（可选上午/下午/全天） |
| **旷工（上午）** | 比设定上班时间晚 **1 小时** 仍未打卡 |
| **旷工（下午）** | 比设定上班时间晚 **1.5 小时** 仍未打卡 |

请假时间窗口规则：
- 上午打卡时间之前请假：可选上午 / 下午 / 全天
- 上午签退后、下午打卡前请假：仅可选下午
- 其他时间不允许请假（程序自动判断）

晚上加班不设迟到、早退、旷工判定，仅记录签到签退时间（鼓励加班）。

### 3.1.1 打卡提醒

APP 在以下时间点发送手机通知提醒：
- 上午上班打卡前 5 分钟
- 上午下班签退时间到
- 下午上班打卡前 5 分钟
- 下午下班签退时间到

### 3.2.1 拍摄日模式

针对自媒体工作性质，支持"拍摄日"特殊模式，与普通办公日完全不同：

**触发方式：**
- 提前规划：在每周开始前预设未来哪天是拍摄日
- 当天切换：在当天上午打卡时间之前，可手动将当天切换为拍摄日

**拍摄模式特点：**
- 主界面显示"拍摄中"状态
- 不需要上下班打卡
- 预先设置拍摄奖励事项（非金额，可以是一顿饭、一杯奶茶等）

**拍摄复盘（当晚 23:00）：**
- 弹出对话框，询问：
  - 拍摄内容是什么？
  - 拍摄地点在哪？
  - 拍摄是否顺利？
  - 有什么感想？
- 系统自动根据回答生成一段文字总结

**拍摄日战报：**
- 与办公日战报不同，展示拍摄信息、复盘总结、奖励金额
- 风格与办公日战报保持一致（手账风）
- 可手动保存到手机相册

### 3.3 奖惩制度

所有金额为虚拟记账，每周一结算，现实中转账兑现。

| 类型 | 规则 | 金额 |
|------|------|------|
| 迟到 | 每次 | 用户自定义 |
| 早退 | 每次 | 用户自定义 |
| 旷工 | 每次 | 用户自定义 |
| 请假 | 不扣钱 | 0 |
| 全勤奖励 | 一周无迟到/早退/旷工/请假 | 用户自定义 |

**男友激励承诺（每日）：**
- 每天上午第一次打卡后，弹出对话框
- 男友输入当日承诺奖励金额（如"今天工作超过 8 小时奖励 50 元"）
- 当天总工作时长 ≥ 8 小时，自动触发奖励，记录在战报和账本中
- 该承诺在主界面长期可见，起激励作用

### 3.4 每日战报

每天工作结束后生成手账风格的精美长图，包含：

- 当天所有打卡时间详情
- 出勤状态判定（正常/迟到/早退/旷工/请假）
- 当天罚款/奖励金额汇总
- 加班时长（如有）
- 当日工作时长统计
- 超过 8 小时的特殊鼓励框（含 emoji 和鼓励语）
- 男友激励承诺及兑现情况
- 当天完成的工作任务清单
- 一句随机鼓励语

格式：排版精美的手账风格长图，可手动保存到手机相册。

### 3.5 当日工作任务清单

嵌入主界面，可在工作期间自由增删改，类似 Todo List。
完成的任务会记录到当天的战报中。

### 3.6 历史记录

三种视图维度：

| 视图 | 展示方式 | 奖惩金额显示 |
|------|---------|------------|
| **周视图** | 时间轴卡片流，每天一张卡片，包含打卡详情、出勤状态、工作时长 | 按天汇总，点击可看明细 |
| **月视图** | 日历格子，颜色标记出勤状态（绿=正常，黄=迟到或早退，红=旷工，蓝=请假，橙色=拍摄日） | 按周汇总，点击可看明细 |
| **年视图** | 12 个月汇总卡片，显示出勤天数、迟到/旷工次数、总时长、总奖惩 | 按月汇总，点击可看明细 |

### 3.7 每周任务对赌

- 每周开始前，用户设定本周具体任务（如"写 3 篇文章、发 5 条视频"）
- 设定完成奖励金额和超额单任务奖励金额
- 设定未完成的惩罚金额
- 周末结算：完成 → 奖励；超额 → 额外奖励；未完成 → 惩罚
- 奖惩金额计入虚拟账本
- 每周一至周日为一个周期，周日晚上自动结算

### 3.8 联网同步

- 每次打卡、生成战报时，通过 WebSocket 实时推送至后端
- 男方通过网页检阅端实时查看女友的工作状态
- 历史记录自动备份至云端，换手机或重装 APP 后可恢复全部数据

### 3.9 设置页面

用户可自定义的参数：

| 参数 | 说明 |
|------|------|
| 上午上班时间 | 如 09:00 |
| 上午下班时间 | 如 12:00 |
| 下午上班时间 | 如 14:00 |
| 下午下班时间 | 如 18:00 |
| 迟到罚款金额 | 每次 |
| 早退罚款金额 | 每次 |
| 旷工罚款金额 | 每次 |
| 全勤奖励金额 | 每周 |
| 对赌基础奖励 | 每周任务完成 |
| 对赌超额奖励 | 每个额外任务 |
| 对赌惩罚金额 | 未完成 |
| 工作日选择 | 自由勾选周一至周日中哪些天为工作日 |
| 拍摄日奖励金额 | 每个拍摄日的奖励金额 |
| 男友奖励时长门槛 | 当日工作时长超过此值触发奖励，默认 8 小时，可自定义 |

### 3.10 首次使用引导

APP 首次打开时，通过逐步引导流程帮助用户完成初始设置：
- 设置上午/下午上下班时间
- 选择工作日（周一至周日自由勾选）
- 设定迟到/早退/旷工的罚款金额
- 设定全勤奖励金额
- 设定男友奖励时长门槛
- 设定拍摄日奖励金额
- 规划未来拍摄日（可选，可跳过）

完成后进入主界面，所有参数后续可在设置页面中修改。

### 3.11 激励功能

**连续出勤天数：**
- 主界面显示"已连续正常出勤 X 天"
- 发生迟到/早退/旷工/请假时，计数归零
- 激励效果类似"连续签到"

**通知栏常驻卡片：**
- Android 通知栏显示常驻卡片，无需打开 APP 即可查看今日状态
- 显示内容：今日已打卡 ✅ / 今日未打卡 ⏳ / 拍摄中 📸
- 点击通知可直接进入 APP

## 4. 旧项目教训与架构改进

上版项目（D:\my-project\work-tracker）存在的问题及本版的改进策略：

| 旧版问题 | 新版改进 |
|---------|---------|
| 所有状态和逻辑集中在一个 App.jsx（436 行） | 按功能模块拆分独立的 Python 文件，每个模块职责单一 |
| 大量重复代码（打卡逻辑、请假逻辑等） | 公共逻辑抽取为工具函数/服务类，杜绝复制粘贴 |
| React state + localStorage 双写同步脆弱 | 统一数据访问层（Repository 模式），所有读写走同一入口 |
| 数据通过 DOM 属性传递（反模式） | 数据通过函数参数 / 对象传递 |
| 组件职责过重，一个组件渲染多套不同 UI | 每个组件只负责一种状态/场景，按状态路由到不同组件 |
| 副作用（通知等）分散在 5+ 个地方 | 通知调度集中在独立模块统一管理 |

**核心原则：单一职责、数据流单向、公共逻辑不重复。**
